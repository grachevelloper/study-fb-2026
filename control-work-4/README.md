# Контрольная работа №4

## Практическое занятие 24: подготовка к контрольной работе №4

Контрольная работа №4 включает результаты выполнения практических заданий 19-23 по дисциплине "Фронтенд и бэкенд разработка".

## Состав работы

```text
practice-19/      # REST API с PostgreSQL
practice-20/      # REST API с MongoDB
practice-21/      # Кэширование с использованием Redis
practice-22-23/   # Балансировка нагрузки и контейнеризация с Docker Compose
```

## Выполненные практики

### Практика 19

Реализовано REST API на Node.js и Express для работы с пользователями. Данные хранятся в PostgreSQL.

Основные возможности:

- создание пользователя;
- получение списка пользователей;
- получение пользователя по id;
- обновление пользователя;
- удаление пользователя.

Запуск:

```bash
cd practice-19
npm install
npm run dev
```

### Практика 20

Реализовано REST API на Node.js и Express для работы с пользователями. Данные хранятся в MongoDB.

Основные возможности:

- подключение к MongoDB через `MONGO_URI`;
- CRUD-операции для пользователей;
- работа с идентификаторами MongoDB.

Запуск:

```bash
cd practice-20
npm install
npm run dev
```

### Практика 21

Реализовано кэширование данных с использованием Redis.

Основные возможности:

- кэширование запросов пользователей;
- кэширование запросов товаров;
- разные TTL для разных типов данных;
- автоматическая инвалидация кэша при изменении данных.

Запуск:

```bash
cd practice-21
cp .env.example .env
npm install
npm run redis:up
npm run dev
```

### Практики 22-23

Реализовано приложение с несколькими backend-сервисами и балансировкой нагрузки через Nginx. Вся система запускается через Docker Compose.

Основные возможности:

- несколько идентичных backend-сервисов на Express;
- Nginx как балансировщик нагрузки;
- Dockerfile для backend-сервиса;
- `docker-compose.yml` для запуска всего стека;
- отдельная Docker-сеть для сервисов;
- настройки отказоустойчивости `max_fails` и `fail_timeout`;
- резервный backend-сервер;
- дополнительный пример балансировки через HAProxy.

Запуск:

```bash
cd practice-22-23
docker compose up --build
```

Проверка балансировки:

```bash
for i in {1..6}; do curl -s http://localhost/; echo; done
```

## Проверка работоспособности

Перед сдачей необходимо проверить каждую практику отдельно:

```bash
cd practice-19
npm install
npm run dev
```

```bash
cd practice-20
npm install
npm run dev
```

```bash
cd practice-21
npm install
npm run redis:up
npm run dev
```

```bash
cd practice-22-23
docker compose up --build
```

Для практик 19-21 проверка выполняется через `curl` по маршрутам, указанным в README соответствующей практики. Для практик 22-23 проверяется чередование ответов от разных backend-серверов при повторных запросах к `http://localhost/`.
