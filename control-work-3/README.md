## Практики 13–14 — PWA и Service Worker

**Папка:** `practice-13-14/`

Статическое PWA-приложение без сервера. Ванильный JavaScript.

### Что реализовано

- **Web App Manifest** — приложение можно установить на устройство (кнопка «Установить PWA»)
- **Service Worker** — перехват сетевых запросов, два режима кэширования:
  - Cache First — для статических ресурсов (CSS, JS, SVG)
  - Network First — для навигационных запросов, с офлайн-страницей (`offline.html`)
- **localStorage** — хранение задач без сервера, данные сохраняются между сессиями
- **Офлайн-режим** — при отсутствии сети приложение работает из кэша
- TODO-менеджер: добавление, удаление, фильтрация (все / активные / выполненные), статистика

### Запуск

Файлы в папке `practice-13-14/` открываются через любой статический сервер.
Service Worker требует `localhost` или HTTPS.

```bash
# Например, через встроенный сервер Python:
cd practice-13-14
python3 -m http.server 8080
# → http://localhost:8080
```

## Практики 15–16–17 — HTTPS, WebSocket, Push и Напоминания

**Папка:** `practice-15-16-17/`

Полноценное full-stack приложение. TypeScript на сервере и клиенте.

### Что реализовано

**Практика 15 — HTTPS + App Shell:**
- Node.js/Express сервер с раздачей статики
- Локальный HTTPS через mkcert (самоподписанный доверенный сертификат)
- Архитектура App Shell: постоянный каркас (`index.html`) + динамический контент (`home.html`, `about.html`) загружаемый через `fetch`

**Практика 16 — WebSocket + Push:**
- WebSocket через Socket.IO — уведомления в реальном времени между вкладками
- Web Push API + VAPID — системные push-уведомления при добавлении задачи
- Подписка/отписка от уведомлений с сохранением подписки на сервере

**Практика 17 — Детализация Push (Напоминания):**
- Форма с полем `datetime-local` для создания задачи с напоминанием
- Серверное планирование: `setTimeout` на нужную задержку, хранилище активных таймеров в `Map`
- Push-уведомление с кнопкой действия «Отложить на 5 минут»
- Service Worker обрабатывает клик по кнопке и делает запрос к `/snooze` даже при закрытой вкладке
- Сервер переносит напоминание: отменяет старый таймер, ставит новый на 5 минут

### Запуск

```bash
cd practice-15-16-17
npm install
npm run build
npm start
# → http://localhost:3001
```

Подробнее — в [`practice-15-16-17/README.md`](./practice-15-16-17/README.md).

## Ключевые концепции

### Progressive Web App (PWA)
Веб-приложение, которое можно установить на устройство и которое работает офлайн. Требует: HTTPS (или localhost), Web App Manifest, Service Worker.

### Service Worker
Отдельный JavaScript-поток браузера. Перехватывает сетевые запросы, управляет кэшем, получает push-уведомления даже при закрытой вкладке.

### App Shell
Архитектурный паттерн: минимальный HTML-каркас кэшируется навсегда, а контент загружается динамически. Даёт мгновенную первую отрисовку и возможность раздельного кэширования.

### Web Push + VAPID
Стандарт для отправки уведомлений через инфраструктуру браузера (серверы Google, Mozilla). VAPID — пара ключей для идентификации сервера-отправителя. Уведомление доходит до Service Worker, который показывает его пользователю.

### WebSocket
Постоянное двустороннее соединение между браузером и сервером. В отличие от HTTP, сервер может отправлять данные клиенту в любой момент без запроса.
